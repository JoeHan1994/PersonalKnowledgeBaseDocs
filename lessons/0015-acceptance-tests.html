自动化验收:四步冒烟,写成一键回归

第 12 课手敲的 curl,这一课固化成 pytest。以后改完一条命令,确认没改坏。

这一课的小胜利:你能用 TestClient 把第 12 课那四步验收 (健康 / 投喂 / 命中 / 库外认怂)写成四个 test_ 函数, 再用依赖覆盖把真模型、真 Chroma 换成假实现—— 测试不连网、毫秒级、每次结果一样。pytest -q 全绿,才敢说"这次改动没破坏老功能"。

先认词 · 本课会反复出现

测试客户端 TestClient
FastAPI 给的"假浏览器":不用真起服务器,直接 client.post("/ingest", json=…) 调你的应用。
断言 assert
写下"我期望什么":assert resp.status_code == 200。不成立就测试失败、亮红。
依赖覆盖 dependency_overrides
测试时把 Depends(...) 里的真实现替成假的——不碰真模型/真向量库,结果才稳定。
假实现 fake / stub
一个最小可控的替身(比如固定返回某向量的 FakeProvider)。让测试确定,不受模型随机性影响。
确定性 determinism
同样输入永远同样输出。测试必须确定,否则今天绿明天红,谁也不敢信。
回归 regression
"改 A 把 B 弄坏了"。回归测试就是那张安全网:每次改完跑一遍,坏了立刻报红。

为什么不直接连真模型测

真 Ollama / 真 Chroma 又慢又"飘"——同一句话,生成的答案可能每次略不同,测试就会时绿时红。 所以验收测试要换上假实现:嵌入返回固定向量、生成返回固定句式。 这样测的是你的编排逻辑(投喂→检索→认怂),而不是模型本身。模型质量另有评测。

TestClient 假浏览器
override 注入假 Provider

pytest -q 一键回归

亲手试:跑一遍测试,再故意改坏看它报红

点运行测试,四个 test_ 逐个变绿。再勾上引入 bug(假装把"认怂阈值"调没了), 重跑——库外那条立刻报红。这就是回归网在干活。(运行是诚实模拟,真命令见下方。)

① conftest.py — 假实现 + 依赖覆盖 + 客户端夹具:

  
# privateknowledgebase.API/tests/conftest.py
import pytest
from fastapi.testclient import TestClient
from app.main import app
from app.interface.http.deps import get_answer_service, get_ingest_service

class FakeProvider:                       # 固定向量,确定性
    def embeddings(self, model, inputs):
        return [[1.0, 0.0, 0.0] for _ in inputs]

@pytest.fixture
def client():
    # 用假 Provider 组出 ingest / answer 服务(内存向量库),替换真实现
    ingest, answer = build_services_with(FakeProvider())   # 你的组装小工厂
    app.dependency_overrides[get_ingest_service] = lambda: ingest
    app.dependency_overrides[get_answer_service] = lambda: answer
    yield TestClient(app)
    app.dependency_overrides.clear()      # 测完还原,别污染别的测试
  

  

    dependency_overrides 是 FastAPI 内建的测试接缝:不改一行业务代码,就能把 Depends(...)
    里的真东西换成假的。这正是第 9–10 课"组合根集中接线"换来的好处。
  


  
② test_acceptance.py — 四步验收,逐条断言:

  
# privateknowledgebase.API/tests/test_acceptance.py
def test_health(client):
    resp = client.get("/health")
    assert resp.status_code == 200
    assert resp.json() == {"status": "ok"}

def test_ingest(client):                                  # 投喂
    resp = client.post("/ingest", json={
        "path": "notes/e.md", "content": "# 嵌入\n嵌入把文本变成向量。"})
    assert resp.status_code == 200
    assert resp.json()["chunks_ingested"] >= 1

def test_ask_in_library(client):                          # 命中:有答案 + 有出处
    client.post("/ingest", json={"path": "notes/e.md", "content": "嵌入把文本变成向量。"})
    resp = client.post("/knowledge/ask", json={"question": "嵌入是什么?", "k": 3})
    assert resp.status_code == 200
    assert resp.json()["sources"]                         # 非空才算命中

def test_ask_off_library(client):                         # 库外:必须认怂
    resp = client.post("/knowledge/ask", json={"question": "明天天气?", "k": 3})
    assert resp.status_code == 200
    assert resp.json()["sources"] == []                   # 没出处,认怂
