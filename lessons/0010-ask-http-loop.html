Lesson 0010 · 个人知识库

闭环:把刚存进去的资料问出来

上一课装了"存"的门,这一课装"问"的门——同一个 Chroma,两头打通。

这一课的小胜利:你能讲清一个 POST /knowledge/ask 请求的读路径—— 问题嵌成查询向量 → Chroma 取 top-k → 拼成带出处的提示词 → 模型生成答案; 并说出"存和问共用同一个向量库实例"才是闭环的关键。

先认词 · 本课会反复出现

读路径 Query path
和上一课"写路径"相对:请求进来不是存,而是查 + 答。同样穿过四层,方向一致、动作相反。
查询向量 Query embedding
把用户问题用同一个嵌入模型变成的向量。只有同模型、同维度,才能和库里的块比距离。
闭环 Closed loop
"存进去"和"问出来"落在同一个 Chroma 上。靠组合根里共享同一个适配器实例实现。
认怂阈值 Threshold
最高分仍低于阈值 ⇒ 库里没有 ⇒ 如实说"不知道"。防幻觉的第一道闸(第 6 课)。
出处 Citation
每条答案带上来源块的 id/元数据,让人能回溯核对。接地(grounding)的凭据。
检索用例 RetrieveContext
第 5 课那只手:嵌问题 → 查库 → 返回带分数的块。问答用例 AnswerQuestion 调它。

同一个 Chroma,两道门

上一课 [L9] 你把文件存进了 ChromaVectorStore。这一课不再新建库——复用同一个实例, 只是反着走:请求带来一个问题,沿四层下沉去查,再把答案逐层返回。

interfacePOST /knowledge/ask 收问题
applicationRetrieveContext:嵌问题 → 查 top-k
applicationAnswerQuestion:拼提示词 → 生成
response答案 + 出处返回

亲手试:对刚入库的资料提问

下面这个"库"已预装三块(就当是上一课 ingest 进去的)。问个问题点 提问: 会逐层亮起读路径,按词面相似度取最像的块,拼出带出处的答案。 问一句库里没有的(如"明天天气?")——会触发认怂阈值,如实说不知道。

就绪

落到你的项目:两块 DTO + 一个路由 + 共享实例

① 请求/响应体(interface):

# app/interface/http/schemas.py  ——  追加
class AskIn(BaseModel):
    question: str
    k: int = 3

class SourceOut(BaseModel):
    id: str
    score: float
    meta: dict

class AskOut(BaseModel):
    answer: str
    sources: list[SourceOut]      # 出处:可回溯

② 路由(interface)——只收发,转交问答用例:

# app/interface/http/routers/knowledge.py
router = APIRouter(prefix="/knowledge", tags=["knowledge"])

@router.post("/ask", response_model=AskOut)
async def ask(
    body: AskIn,
    service: AnswerQuestion = Depends(get_answer_service),   # 第 6 课的用例
) -> AskOut:
    result = await service.run(body.question, body.k)
    return AskOut(
        answer=result.text,
        sources=[SourceOut(id=s.id, score=s.score, meta=s.meta) for s in result.sources],
    )

③ 接线(组合根)——闭环的关键:同一个 store 实例喂给两个用例:

# app/interface/http/deps.py  ——  Container.__init__
store = ChromaVectorStore(settings.chroma_path)        # ← 只造一次

# 写路径(上一课)
self.ingest = IngestDocument(loaders, chunker, self.chat, store, settings.embed_model)
# 读路径(这一课)
retrieve = RetrieveContext(self.chat, store, settings.embed_model)   # 第 5 课
self.answer = AnswerQuestion(retrieve, self.chat)                    # 第 6 课

def get_answer_service() -> AnswerQuestion:
    return _container.answer

ingest 和 answer 拿到的是同一个 store—— 所以你 POST /ingest 存的东西,POST /knowledge/ask 立刻能查到。这就是闭环。

跑通它(服务在 8001):先存再问。
curl -X POST http://localhost:8001/knowledge/ask -H "Content-Type: application/json" -d '{"question":"嵌入是什么?","k":3}'
