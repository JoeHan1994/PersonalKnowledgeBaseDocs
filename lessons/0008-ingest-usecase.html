IngestDocument:第一次把你的文件喂进库

加载器、分块器、嵌入、向量库——这一课把它们串成一个用例。

这一课的小胜利:你能说清一个 IngestDocument 用例怎样 编排"加载 → 分块 → 嵌入 → 入库"四步;并亲手验证:同一个文件反复入库不产生副本 (靠稳定 id + upsert),改一处只更新那一处。

先认词 · 本课会反复出现

用例 Use case
应用层里"一件完整的事"的入口,如IngestDocument(把一份资料收进库)。它只编排步骤,不自己干脏活。
编排 Orchestration
按顺序调用各个部件(加载器→分块器→嵌入→向量库),把它们的输入输出接起来。用例的本职工作。
稳定 id Deterministic id
由内容来源算出的固定编号,如 note.md#0。同一块每次都得同一个 id,这样才能"认得出旧的"。
幂等 Idempotent
同样的输入跑一次和跑十次,库里的结果一样。靠稳定 id 实现——重复入库不堆副本。
upsert update-or-insert
有则覆盖、无则新增。向量库按 id 入库时用它,保证改一处只换一处。
批量 Batch
把一批块一次性算嵌入、一次性写库,比逐个来快得多——嵌入和写库都喜欢成批。

落到你的项目:用例长什么样

端口都已就位:加载器 Loader(第 7 课)、向量库 VectorStore(第 4 课)、 嵌入 LLMProvider.embeddings(model, inputs)(你代码里已有)。用例只是把它们编排起来: 

# app/application/ingest_service.py  ——  application 层,只编排
class IngestDocument:
    def __init__(self, loaders, chunker, embedder, store, model):
        self._loaders = loaders      # 扩展名 -> Loader(第 7 课注册表)
        self._chunker = chunker      # Document -> list[Chunk](第 2 课)
        self._embedder = embedder    # LLMProvider(你已有)
        self._store = store          # VectorStore(第 4 课)
        self._model = model          # 如 "nomic-embed-text"(本地 Ollama)

    async def execute(self, path: str) -> int:
        loader = self._loaders.for_path(path)     # 按扩展名挑加载器
        doc = loader.load(path)                    # 异构 -> 统一 Document
        chunks = self._chunker.split(doc)          # 一份 -> 多块
        ids   = [f"{doc.id}#{i}" for i, _ in enumerate(chunks)]   # 稳定 id
        texts = [c.text for c in chunks]
        vectors = await self._embedder.embeddings(self._model, texts)  # 批量
        self._store.upsert(ids, vectors, chunks)   # 有则覆盖、无则新增
        return len(chunks)

三个细节决定它好不好用:(1) 稳定 id 让重复入库幂等; (2) 批量 把一份文件的所有块一次算、一次写; (3) 用例不 import 任何适配器——只认端口,依赖始终向内。
