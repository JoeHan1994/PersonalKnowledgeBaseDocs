重排 re-rank:先粗筛一批,再精挑前几

向量召回又快又糙;加一道交叉编码器精排,把真正相关的那条顶上来。

这一课的小胜利:你能在第 5、10 课的检索之上,插一道重排: 先用向量库召回一批候选(比如 20 条),再用交叉编码器对 (问题, 候选) 逐对精排,只把最相关的前 3 条喂给生成。 答案的相关性肉眼可见地变好,而读路径只多挂一个适配器。

先认词 · 本课会反复出现

双编码器 bi-encoder
问题、文档各自编码成向量再比距离(就是你现在的嵌入检索)。快,能扫海量,但略糙。
交叉编码器 cross-encoder
把 (问题, 文档) 一起喂进模型,直接吐一个 0–1 相关分。准得多,但每对都要算一次,慢。
召回 retrieve / recall
第一阶段:向量库快速捞出一批可能相关的候选(top-N,宁多勿漏)。
精排 re-rank
第二阶段:只对那一批候选,用交叉编码器细评,重新排序,留 top-k。
候选集 candidate set
召回出的那 N 条。交叉编码器太贵,只能在这一小批上跑,不能对全库跑。
二阶段检索 two-stage
"快而广的召回 + 慢而准的精排"配合:用便宜的先缩小范围,把贵的算力花在刀刃上。

为什么要两道工序

向量检索(双编码器)是各自编码再比距离,能秒扫整库,但有时把"字面像、其实不相关"的也排前面。 交叉编码器把问题和候选放一起读,判断更准——可它每对都要算,扫全库太慢。 所以套路是:先召回 20 条,再精排出 3 条。便宜的缩范围,贵的提精度。

      ② 交叉编码器适配器(infrastructure)

      # app/infrastructure/rerank/cross_encoder.py
from sentence_transformers import CrossEncoder   # pip install sentence-transformers

class CrossEncoderReRanker:
    def __init__(self, model: str = "cross-encoder/ms-marco-MiniLM-L-6-v2"):
        self._model = CrossEncoder(model)        # 本地模型,离线可跑

    def rerank(self, query: str, candidates: list[Chunk]) -> list[Chunk]:
        pairs = [(query, c.text) for c in candidates]
        scores = self._model.predict(pairs)      # 每对一个 0~1 相关分
        ranked = sorted(zip(candidates, scores), key=lambda x: x[1], reverse=True)
        return [c for c, _ in ranked]

      ③ 读路径改两阶段(application) —— 召回放大、精排收窄:

      # app/application/retrieve_context.py
class RetrieveContext:
    def __init__(self, store: VectorStore, reranker: ReRanker):
        self._store = store
        self._reranker = reranker

    def execute(self, query: str, k: int = 3, fetch: int = 20) -> list[Chunk]:
        candidates = self._store.search(query, k=fetch)   # 1) 召回放大到 20
        ranked = self._reranker.rerank(query, candidates) # 2) 交叉编码器精排
        return ranked[:k]                                 # 3) 只留前 k 条
