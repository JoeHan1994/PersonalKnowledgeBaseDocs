Lesson 0003 · 个人知识库

嵌入:把"块"变成可比较的向量

"相似"不是玄学,它就是两个向量夹角的余弦。

这一课的小胜利:你能亲手转动两个向量、实时读出它们的余弦相似度, 并说清"为什么把向量放大一倍、相似度却纹丝不动"——然后认出你项目里 domain 已有的 LLMProvider.embeddings() 端口, 就是把"块"变成向量的入口。

先认词 · 本课会反复出现

嵌入 Embedding
把一段文字交给"嵌入模型",换回一串数字的动作。这串数字就是它的语义坐标。
向量 Vector
就是那一串数字,例如 384 或 768 个小数。可以想成高维空间里的一个带方向的箭头。
维度 Dimension
向量里有几个数字,就是几"维"。768 维 = 768 个数字一起描述这段话的含义。
余弦相似度 Cosine similarity
两个向量夹角的余弦值,范围 −1~1。只看方向不看长度,所以对文本长短不敏感。本课主角。
点积 Dot product
两向量对应数字相乘再相加。方向 + 长度都算;若先归一化,它就等于余弦。
归一化 Normalize
把向量长度统一拉成 1,只保留方向。归一化后,余弦/点积/欧氏排序基本一致。
端口 / 适配器 Port / Adapter
端口=领域层声明"我要个能算向量的东西"(契约);适配器=基础层用 Ollama/远程 API 真正去算。

嵌入到底做了什么?

回忆离线管道:[L1] 加载 → 分块 → 嵌入 → 入库。上一课你把长文切成了"块" [L2];这一课,要把每个块交给嵌入模型, 换回一串数字——一个向量(例如 384 或 768 个浮点数)。

神奇之处:意思相近的文本,向量在空间里方向相近。 于是"语义相似"这件模糊的事,被翻译成了一道能精确计算的几何题。 检索时不再靠关键词匹配,而是问:"哪些块的向量,方向最贴近问题的向量?"

怎么量"方向有多近"?余弦相似度

答案是余弦相似度:两个向量夹角的余弦值。[1] 范围 −1 到 1——1 完全同向、0 正交无关、−1 完全反向。 关键性质:它只看方向、不看长度,所以对文本长短不敏感。亲手转一转 👇

拖动两条长度滑块——余弦值不动;只有转动方向它才变。 这就是检索不怕"长块 vs 短块"的原因:比的是方向,不是大小。

那道几何题长这样

余弦相似度的公式,就是"点积 ÷ 两个模长":

cos(A, B) = (A · B) / (‖A‖ × ‖B‖)

其中点积 A · B = Σ aᵢbᵢ,模长 ‖A‖ = √(Σ aᵢ²)。 维度从 2 升到 768,公式一字不改——你在上面玩的就是真东西,只是看得见而已。

落到你的项目

好消息:把文本变成向量的端口已经在你项目里了。domain 定义"要什么",infrastructure 决定"谁来做"(本地 Ollama / 远程 API):

# app/domain/ports.py(领域层 · 只声明契约)
class LLMProvider(Protocol):
    async def embeddings(self, model: str, inputs: list[str]) -> list[list[float]]: ...

# app/domain/services.py(领域层 · 纯数学,无 I/O)
def cosine(a: list[float], b: list[float]) -> float:
    dot = sum(x * y for x, y in zip(a, b))
    na = math.sqrt(sum(x * x for x in a))
    nb = math.sqrt(sum(y * y for y in b))
    return dot / (na * nb)

记住这条分界:"调模型算向量"是 I/O,归 infrastructure(Ollama 适配器); "算余弦"是纯数学,归 domain。 下一课你会把这些向量连同块一起塞进 Chroma,让"找最近的方向"这件事变得飞快。
