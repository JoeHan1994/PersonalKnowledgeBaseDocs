先认词 · 本课会反复出现

联调 Integration
把各自写好的前端、后端、Chroma 接到一起,确认它们能合作,而不只是各自单测能过。
跨域 CORS
浏览器规矩:3000 的页面访问 8001 的后端属"跨域",后端要表态允许。你项目已配 CORSMiddleware。
来源 Origin
协议+域名+端口三者合一。localhost:3000 和 localhost:8001 是不同来源——这才需要 CORS。
健康检查 Health check
GET /health 返回 200,确认后端活着。联调第一步,排除"根本没起来"。
冒烟测试 Smoke test
用几条最关键的请求快速验主流程通不通,不求覆盖全面,只求"没冒烟(没炸)"。
验收 Acceptance
站在用户角度跑真实场景:存得进、问得出、库外认怂。过了才算这个切片真的完成。

联调前,补最后一处接线

第 9–10 课写的 ingest、knowledge 路由,还没注册进应用。 联调第一件事就是在 main.py 把它们挂上——否则请求会 404。 好消息:CORS 你早就配好了(allow_origins=["*"]),3000 直接能访问 8001。 

# app/main.py  ——  把新路由挂上(否则 /ingest、/knowledge/ask 都 404)
from .interface.http.routers import agent, chat, ollama, providers, ingest, knowledge

app.include_router(ingest.router)       # 第 9 课
app.include_router(knowledge.router)    # 第 10 课

# CORS 已存在,无需改动:
# app.add_middleware(CORSMiddleware, allow_origins=settings.cors_origins,  # 默认 ["*"]
#                    allow_credentials=False, allow_methods=["*"], allow_headers=["*"])

为什么 allow_credentials=False 能配 ["*"]?CORS 规矩:通配 "*" 与 "带凭据(cookie)"不能并存。你这里不带凭据,所以 ["*"] 合法又省事。

点 运行验收,逐条亮起(这是验收清单模拟,每条都对应下方一条真实命令)。 四条全绿,就是你这套切片的毕业证。

就绪

真实命令:照着敲一遍

开两个终端,分别起服务:

# 终端 A — 后端(在 privateknowledgebase.API 目录)
pip install chromadb            # 还没装的话
uvicorn app.main:app --reload --port 8001

# 终端 B — 前端(在 privateknowledgebase.Web 目录)
npm run dev                     # http://localhost:3000

然后用 curl 跑四步冒烟(或直接在 3000 的页面点投喂/提问):

# 1) 后端活着?
curl http://localhost:8001/health
#   => {"status":"ok"}

# 2) 投喂一份文件
curl -X POST http://localhost:8001/ingest -H "Content-Type: application/json" \
  -d '{"path":"notes/embeddings.md","content":"# 嵌入\n嵌入把文本变成向量。"}'
#   => {"path":"notes/embeddings.md","chunks_ingested":1}

# 3) 提问 —— 应拿到答案 + 出处
curl -X POST http://localhost:8001/knowledge/ask -H "Content-Type: application/json" \
  -d '{"question":"嵌入是什么?","k":3}'
#   => {"answer":"...","sources":[{"id":"notes/embeddings.md#0", ...}]}

# 4) 库外问题 —— 应如实认怂
curl -X POST http://localhost:8001/knowledge/ask -H "Content-Type: application/json" \
  -d '{"question":"明天天气怎么样?","k":3}'
#   => {"answer":"我在库里没找到相关内容…","sources":[]}

Chroma 把数据写到 settings.chroma_path(本地目录)。把它加进 .gitignore, 别把向量库提交进仓库。重启后端,数据还在——这就是持久化(第 9 课)。
