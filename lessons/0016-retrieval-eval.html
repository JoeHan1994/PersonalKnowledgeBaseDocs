检索评测:别凭感觉,用数字说话

攒一个小黄金集,算命中率/MRR/召回率;重排到底提了多少,一目了然。

这一课的小胜利:你能攒一个黄金集(几十条"问题 + 正确出处"), 写三个小函数算 命中率@k / MRR / 召回率@k, 然后把"只检索"和"检索+重排"(第 14 课)各跑一遍—— 指标涨了多少看得见。从此调参不再靠玄学,靠数字。

先认词 · 本课会反复出现

黄金集 golden set
人工标好的"标准答案":每个问题配上哪些 chunk 才算正确出处。评测的尺子,质量决定一切。
命中率@k Hit / Accuracy@k
前 k 条里只要有一条正确,就算这题命中。统计命中的题占比。最直观的"找没找到"。
MRR Mean Reciprocal Rank
平均倒数排名:正确项排第 1 得 1 分、第 2 得 1/2、第 3 得 1/3…取平均。越靠前越值钱。
召回率@k Recall@k
前 k 条里捞回了几成该捞的正确出处。一题有多个正确答案时尤其看它。
离线评测 offline eval
不打扰用户,拿固定黄金集批量跑出分数。改一版就重测一版,可对比、可复现。
基线对比 A/B baseline
把"老做法(只检索)"当基线,新做法(加重排)和它比。涨了才留,没涨就别加复杂度。

三把尺子,量同一件事的不同侧面

命中率问"找到没"(粗),MRR问"排得够不够靠前"(在乎名次), 召回率问"该找的捞回几成"(在乎全不全)。 重排不增删候选、只重新排序,所以它最该让 MRR 涨——把对的顶到前面。

② 三把尺子 — 纯函数,给一题的"命中 id 排序"打分:

# tests/eval/metrics.py
def hit_at_k(ranked_ids, relevant, k):
    return 1.0 if set(ranked_ids[:k]) & relevant else 0.0

def reciprocal_rank(ranked_ids, relevant):
    for i, cid in enumerate(ranked_ids, start=1):
        if cid in relevant:
            return 1.0 / i          # 第 1 名→1.0,第 2 名→0.5 …
    return 0.0

def recall_at_k(ranked_ids, relevant, k):
    hits = set(ranked_ids[:k]) & relevant
    return len(hits) / len(relevant)   # 该捞的捞回几成

③ 评测跑批 — 同一黄金集,基线 vs 重排各跑一遍:

# tests/eval/run_eval.py
def evaluate(retrieve, k=3):
    hit = mrr = rec = 0.0
    for item in GOLDEN:
        ranked = [c.id for c in retrieve(item["q"], k=10)]   # 你的读路径
        hit += hit_at_k(ranked, item["relevant"], k)
        mrr += reciprocal_rank(ranked, item["relevant"])
        rec += recall_at_k(ranked, item["relevant"], k)
    n = len(GOLDEN)
    return {"hit@%d" % k: hit/n, "mrr": mrr/n, "recall@%d" % k: rec/n}

base = evaluate(retrieve_without_rerank)   # 基线
ours = evaluate(retrieve_with_rerank)      # 第 14 课的重排
print(base, ours)                          # 涨了才留重排
