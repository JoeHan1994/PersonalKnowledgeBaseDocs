Lesson 0006 · 个人知识库

RAG 问答:知识库第一次"会答话"

把检索和生成串起来——你的第一个端到端垂直切片。

这一课的小胜利:你能输入一个问题,看着系统检索 → 拼提示 → 生成 一路跑完,给出一句带 [1][2] 出处的回答;并亲眼验证"资料里没有就老实说不知道"。 然后把这套串成你项目 application 的 AnswerQuestion 用例。

先认词 · 本课会反复出现

RAG 问答 RAG QA
"检索 + 生成"合体:先查回相关原文,再让模型据此作答。本课把前五课拼成一条龙。
生成 Generation
把"资料 + 问题"交给大模型,让它写出自然语言回答的那一步。用你项目已有的 LLMProvider。
接地 / 有据 Grounding
让回答只基于检索到的资料,而不是模型瞎编。靠提示词约束 + 出处标注实现。
幻觉 Hallucination
模型一本正经编造没有依据的内容。RAG + "没依据就说不知道"正是为了压住它。
垂直切片 Vertical slice
打通"从输入到输出"的一条最小完整链路。先窄而通,再加宽。这一课就是第一刀。
用例 Use case
应用层里一个完整业务动作(这里=回答一个问题)。编排各端口,产出结果。

把前五课接成一条龙

建库那半边你已经懂了(分块 · 嵌入 · 入库); 问答这半边只差最后一步——生成。完整链路:

嵌问题问题→向量
检索取 top-k 块
拼提示资料+问题+约束
生成模型带出处作答

前三步就是上一课的检索 [L5]; 这一课补上第四步,并强调一件事:让模型"有据才说,无据认怂"。

亲手跑一遍端到端

关键:让它"无据认怂"

RAG 不是万能咒。如果检索没取到相关资料,正确做法是回答"不知道", 而不是让模型硬编(那叫幻觉)。两道防线:

· 提示词约束:"只根据资料回答;资料里没有就说不知道。"
· 分数门槛:最高相似度低于某个阈值,就直接判定"库里没有",根本不调用模型。

落到你的项目(里程碑)

这是你的第一个垂直切片。把上一课的检索,接上你早就有的 LLMProvider,组成一个 application 用例:

# app/application/answer_question.py(应用层 · 编排检索 + 生成)
class AnswerQuestion:
    def __init__(self, retrieve: RetrieveContext, llm: LLMProvider): ...
    async def run(self, question: str, k: int = 3) -> Answer:
        chunks = await self.retrieve.run(question, k)  # 上一课
        if not chunks: return Answer(text="不知道", sources=[])
        prompt = build_prompt(question, chunks)  # 带出处约束
        text = await self.llm.chat(prompt)
        return Answer(text=text, sources=[c.meta for c in chunks])

再加一个 interface 路由 POST /knowledge/ask 调它, 切片就贯通了:HTTP 进 → 检索 → 生成 → 带出处的 JSON 出。 这之后,0007+ 才开始横向加各种 Loader(PDF、网页、代码…)把库喂厚。
