分块 Chunking

压缩自 Lesson 0002,设计用于快速查阅与打印。

一句话

块 = 知识库里检索和引用的最小单位。块的质量决定检索的质量。

固定大小 + 重叠

参数含义经验区间
chunk_size每块的长度几百字符(一两段话)
overlap相邻块重叠的长度chunk_size 的 10%–20%
step窗口每次前移 = size − overlap由上两者导出

两端的代价

极端后果
块太大一个向量背负多主题 → 检索不精准、噪声多
块太小语义被切碎 → 单块缺上下文,模型读不懂
零重叠跨边界的关键句两块都不完整 → 检索可能两块都漏

参考算法(固定大小 + 重叠)

step = max(1, chunk_size − overlap)
for start in range(0, len(text), step):
    end = min(start + chunk_size, len(text))
    yield text[start:end]
    if end == len(text): break

注:这是字符级"固定大小"切法,最简单、可预测。进阶有递归切分 (优先在段落/句子/标点等自然边界断开)与按结构切分(代码按函数、Markdown 按标题)。 先用固定大小打通闭环,后续再按数据源升级。

核心原则

沿着每种格式天然的结构边界切;切不动了再退回固定大小+重叠。别对所有数据用同一把刀。

1. 结构切分(标题/函数/幻灯片/说话人) → 2. 递归细分(过大块按段落→句子→标点) → 3. 固定大小兜底(size+overlap)

按数据类型的切割策略 & 可运行示例

每行对应一个可独立运行的脚本(仅标准库、自带样例输入),位于 reference/chunking-examples/。运行:python <脚本>。

数据类型推荐切法天然出处可运行脚本
Markdown / 纯文本按标题递归切,过长段落兜底heading_path markdown_text.py
PDF / Word / PPT按页/幻灯片,固定大小兜底page documents_pdf.py
网页 / 文章抽正文(去 nav/footer/script),按标题+段落url web_article.py
代码仓库AST 按函数/类整体切symbol + 行号 code_repo.py
图片 / 扫描件(OCR)OCR 版面块,过滤低置信/过短page + bbox image_ocr.py
音频 / 视频(转写)按时间窗口/说话人轮次[mm:ss] audio_video.py
聊天记录 / 邮件按时间窗口聚成话题块,非单条conversation + 时间 chat_email.py
混合类型(一源多种内容)先切成带类型片段,再分发到各自切法content_type + 各自出处 mixed_source.py

八个脚本共享同一个纯数据 Chunk(document_id / index / text / meta);切法各异,但产物统一。 需要外部库的格式(PDF/OCR/转写)用 stub 桩函数提供样例输入,因此零依赖即可跑通切分逻辑; 脚本顶部注释写明真实场景该替换成哪个库(pypdf / pytesseract / faster-whisper…)。 一次跑全部:Get-ChildItem reference\chunking-examples\*.py | %{ python $_.FullName }

混合类型是常态(Markdown 里夹代码块/表格、notebook、网页带代码示例): 用组合 / 分发式 Chunker——先把来源切成"带类型的片段", 散文走固定大小、代码走 AST、表格整张一块绝不切碎, 每块在 meta 里记 content_type 以便检索时按类型过滤。见 mixed_source.py。

落到项目:Chunk 在 domain,一个 Chunker 端口; 每个脚本的策略对应 infrastructure 的一个适配器,由 Loader 按文件类型选择 (.md→Markdown、.py→Code、其余→FixedSize 兜底)。加新数据源 = 插一个新适配器。
