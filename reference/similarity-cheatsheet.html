相似度 / 距离度量速查

余弦只是其中一种。挑度量,先看你的数据长什么样。

一句话:度量是"像不像"的尺子。文本语义检索默认用 余弦 / 点积 / 欧氏(归一化后三者排序基本一致);标签用杰卡德、 指纹去重用汉明、打分推荐用皮尔逊、精确词用 BM25 词袋。 全部可跑代码见 all_metrics.py。

八种度量一览

度量量什么 · 范围通俗例子典型场景
余弦
Cosine		 方向夹角,不看长度
[−1, 1]		 口味"比例"一样就高分——饭量大一倍也不影响 语义检索默认;文本长短不敏感(RAG 检索块)
点积 / 内积
Dot / IP		 方向 + 长度都算

欧氏(L2)

[0, ∞) 越小越近		 地图上谁离我直线最近,谁就最像 向量检索;Chroma 默认;归一化后与余弦排序一致

[0, ∞)		 不能走斜线,把每个维度差距相加 高维稀疏数据;对个别极端维度比欧氏更稳健

为什么语义检索里它们经常"差不多"?

一个常被忽略的事实:当向量被归一化(长度都拉成 1)后, 余弦、点积、欧氏给出的排序几乎一致——因为这时 欧氏² = 2 − 2·余弦,余弦越大欧氏越小,是同一回事的不同写法。 所以多数向量库(含 Chroma)默认 L2,你照样能拿到"按语义最近"的结果。

落到你的项目

度量分两类住处:纯数学的算法(余弦/欧氏…)是无 I/O 的,归 domain 的 services.py; 而"用哪种度量做实际检索"是向量库的能力,由 infrastructure 的 VectorStore 适配器配置。 Chroma 创建集合时用 hnsw:space 选 l2(默认)/ cosine / ip—— 这正是第 4 课要做的事。

# 第 4 课会这样建集合(度量 = 一个配置项,不是核心逻辑)
collection = client.create_collection(
    name="kb",
    configuration={"hnsw": {"space": "cosine"}},  # 或 l2(默认) / ip
)
